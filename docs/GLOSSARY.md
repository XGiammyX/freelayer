# Glossary

[← Docs Index](README.md) · [Plain English overview](PUBLIC_EXPLANATION.md)

> [!TIP]
> New to the project? Read [FreeLayer in plain English](PUBLIC_EXPLANATION.md) first — this table is the reference version.

Every FreeLayer term: what it means simply, what it means technically, and whether it exists yet.

| Term | Simple explanation | Technical explanation | Status |
| --- | --- | --- | --- |
| **Capsule** | A sealed digital envelope carrying one update between devices | Versioned encrypted envelope with algorithm identifiers, opaque routing hint, padded payload — the only cross-device format ([ADR-0003](adr/ADR-0003-capsules-as-only-cross-device-format.md), [CAPSULENET.md](CAPSULENET.md)) | Type placeholders only; wire format blocked by Gate E |
| **Blind Courier** | Any delivery channel that carries the sealed envelope without reading it | Untrusted `Transport` implementations (relay, file, QR, USB, LAN, future Tor/radio) carrying opaque ciphertext with per-transport leakage labels ([NETWORK_MODEL.md](NETWORK_MODEL.md)) | Interfaces + test-only mock; no real transport |
| **Sovereign Room** | A private workspace that lives on members' devices, not on a company server | Local-first, event-sourced room holding typed objects (messages, docs, tasks, decisions…), synced via capsules, no homeserver ([SOVEREIGN_ROOMS.md](SOVEREIGN_ROOMS.md), ADR-0006) | Type scaffolding; implementation blocked by Gate H |
| **RoomOS** | The idea that a room is a small private operating space, not just a chat | The room object model + operation log + policy that turns conversations into operational spaces | Local foundation implemented (TECH-16…20): events + replay, object model, privacy-safe queries, and unverified membership + non-authoritative capability scaffolding — all local, memory-only, policy-gated; no sync/crypto/identity |
| **Room membership (placeholder)** | A local note that someone is in a room — not a verified account | Local, UNVERIFIED `RoomMembershipRecordV1` with a placeholder role; authorizes nothing by itself (TECH-20) | Local scaffolding; verified identity is Gate G |
| **Capability descriptor** | A description of what a member may attempt — not a key or token | Non-authoritative `RoomLocalCapabilityDescriptorV1`, bound to room+membership+revision, attenuable, never serialized/persisted; only a `PolicyDecision` authorizes (TECH-20) | Scaffolding; a real capability runtime is Gate B |
| **Local revocation** | Turning off someone's local access when their membership/role/policy changes — on this device only | Membership/policy/mode changes bump a local revision fence; `assertPreparedRoomAuthorizationCurrentV1` revalidates before every side effect (TECH-21) | Local only; **not** distributed revocation (Gate H) |
| **Prepared authorization** | A checked-but-not-yet-executed permission that must be re-checked at the moment of use | Non-authoritative, transient `PreparedRoomAuthorizationV1` bound to the operation + a revision fence; never a token/cache (TECH-21) | Local; single-use/nonce-bound decisions are Gate B |
| **Policy Engine** | The rule system that decides whether the app may store, send, preview, copy or show something | Core pipeline (validate → classify → resolve → strictest wins → `PolicyDecision` → execute → audit); side-effect modules reject calls without a decision ([ADR-0002](adr/ADR-0002-core-enforced-policy-engine.md)) | Types + fail-closed placeholder; real engine is Gate B |
| **Privacy Mode** | A protection level for your whole device: Standard, Private, Ghost, Bunker, Offline Capsule, Emergency, Sovereign Room | Policy profiles over 13 capabilities, resolved strictest-wins ([PRIVACY_MODEL.md](PRIVACY_MODEL.md)) | Schema typed; enforcement is Gate B |
| **Identity Firewall** | No phone number, email or central account required — ever | Local/ephemeral identities, per-contact aliases, one-time invites, QR verification, recovery kit; no server-side user database | Design only (Gate G) |
| **Metadata Firewall** | Fewer silent leaks: typing, read receipts, online status, link previews | Policy-gated application signals + envelope/padding discipline + per-transport leakage labels ([METADATA_MODEL.md](METADATA_MODEL.md)) | Design + 10 defined invariants; tests come with transports |
| **Storage Firewall** | The app can't quietly save things it shouldn't | The write barrier: all persistence flows through StoragePolicy with a `PolicyDecision`; memory-only and null backends exist ([STORAGE_MODEL.md](STORAGE_MODEL.md), ADR-0005) | Interfaces + memory/null providers with runtime guards |
| **Network Firewall** | The app can't quietly talk to the internet | The network side-effect barrier: no connection without NetworkPolicy + mode approval + PBOM entry ([NETWORK_MODEL.md](NETWORK_MODEL.md)) | Design; Gate D |
| **ScreenShield** | Future protection for screenshots, screen recording, clipboard and risky devices | Endpoint Defense Layer: protection levels (off→bunker), capture-aware redaction, platform secure surfaces with honest capability reporting ([SCREENSHIELD.md](SCREENSHIELD.md), ADR-0012) | Research/design only |
| **ProtectedContent** | The only door sensitive content is allowed to appear through | Policy-checked rendering surface (`contentRef` + `PolicyDecision`); raw rendering of sensitive fields is a defect ([PROTECTED_CONTENT_POLICY.md](PROTECTED_CONTENT_POLICY.md)) | Contract designed; not implemented |
| **Ghost Vault** | Future mode keeping your identity keys on an offline device | Cold-identity architecture: the online client couriers encrypted capsules without holding private identity keys | Future design; architecture reserves space |
| **Local AI** | Optional future AI that runs only on your device and is off by default | Policy-gated local inference (summaries, task extraction, Q&A) behind AIPolicy + AI Privacy Guard; zero network calls ([LOCAL_AI.md](LOCAL_AI.md), ADR-0007) | Interfaces + always-rejecting provider |
| **PBOM** | A public list of everything the app actually does with your data | Privacy Bill of Materials: endpoints, storage, permissions, caches, logs, AI behavior — anything missing from it is a bug ([PBOM.md](PBOM.md)) | Live document; automation planned |
| **Threat Model** | The honest list of who could attack you and what FreeLayer can/can't do about it | Assets, attacker classes, in/out of scope, per-component risks ([THREAT_MODEL.md](THREAT_MODEL.md)) | Live document |
| **Trust Center** | The one page that answers "can I trust this today?" (currently: no) | Verified maturity/status per area, updated only when verified reality changes ([TRUST_CENTER.md](TRUST_CENTER.md)) | Live document |
